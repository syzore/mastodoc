# Mastodoc: Documentation Architecture & Guidelines

## 1. The Core Philosophy
Mastodoc is an AI-native, skill-based documentation system. Documentation is a **first-class citizen**. A code task is not "done" until the Knowledge Tree is synced with the codebase.

## 2. The 3-Level Hierarchy
You MUST maintain this structure strictly. Never exceed 3 levels of depth.

### Level 1: The Root (MASTODOC.md)
*   **Location:** `docs/MASTODOC.md`
*   **Purpose:** The Master Routing Table. 
*   **Content:** Contains high-level summaries of major code categories (6-10 total) and links directly to Level 2 Category Branches.

### Level 2: Category Branches (`docs/categories/*.md`)
*   **Location:** `docs/categories/`
*   **Purpose:** Contextual Hubs.
*   **Content:** A 1-paragraph overview of the category and a table of contents (links) to the specific feature files (Level 3) within this domain.

### Level 3: Feature Leaves (`docs/features/[category]/[feature.md]`)
*   **Location:** `docs/features/`
*   **Purpose:** The "Meat."
*   **Content:** Logic, design decisions (ADRs), ideas, and **direct code pointers**.
*   **Rule:** Every Leaf MUST contain relative links to source files (e.g., `[Auth Service](app/api/auth.py)`). 
## 3. Mastodoc Resync Protocol
After any code modification, you MUST execute this protocol before finalizing:

1.  **Traverse:** Identify the specific Feature Leaf (Level 3) that governs the code you just changed.
2.  **Sync:** 
    *   Update logic descriptions and verify relative file links.
    *   Prune resolved TODOs.
    *   Add the human's train of thoughts and decisions made out of which possible options and why.
    *   If a structural change occurred, update the Category Branch (Level 2) and/or create a new ADR.
3.  **Index:** If you added a new Feature Leaf, ensure it is linked in the corresponding Category Branch.
4.  **Constraint:** Do not rewrite unchanged files. Only update the specific nodes (Leaf, Category, or Root) affected by your changes.
5.  **Report:** Provide a one-sentence summary of the documentation updates made.
